# Architecture gap sweep — analysis input (not a decision record)

**Provenance:** 9-lens parallel sweep + synthesis (2026-06-27), evaluating ADR 0003/0004 against architectural patterns to surface gaps. **Status:** input for ADR decisions, not itself authoritative. **Headline:** no model-breaker — the producer/voxel/three-tier spine holds; gaps are additions, not rewrites.

---

# GAP SWEEP SYNTHESIS

## Prioritized gaps

### 1. [blocker] (kit/agent(0004)) — raised by Archetypes, Repetition, Structural, Ornament
**Gap:** Arbitrary-angle / radial / path repetition of REUSABLE parts (N copies at 360/N or tangent-framed angles: pagoda eaves, amphitheater arch rings, rotunda colonnades, polygonal apses, curved arcades, windmill sails). The 24-orientation lattice cannot express the per-instance angles, yet baking one bespoke producer per structure destroys the part/instance/connector/sculpt reuse the whole stack assumes. Neither tier owns 'reusable part, arbitrary repeated angle.'
**Rec:** Add a parametric PATTERN/REVOLVE producer (LinearArray/RadialArray/PathArray) that voxelizes N copies of a sub-geometry at arbitrary angles FROM THE FIELD (lossless parametric tier, same trick angled bastions use) WHILE exposing stable per-element connectors and a member index, so joints/diagnostics can still address 'arch[k].springline.' This hybrid (producer that is also addressable like an instance set) is the single most load-bearing missing primitive; it simultaneously resolves radial symmetry, curved arcades, helical runs, and is the substrate for per-member overrides.

### 2. [major] (kit/agent(0004)) — raised by Repetition, Archetypes, CAD/BIM
**Gap:** Pattern as a first-class EDITABLE construct, not agent-enumerated instances: edit-by-count, per-member override/suppression that survives count/pitch change, 2D grids of openings, repeat a whole bay sub-assembly, repetition-with-variation (index->param), and a 'broken symmetry' diagnostic. The design deliberately pushes all instance repetition onto the LLM, so the scene stores N unrelated instances+joints with no record they are one family.
**Rec:** Make the pattern producer above own its members with stable member-index->NodeId identity, allow the source to be a composite sub-assembly, let HostedOn features ride the array (each member born with its penetrations/sills/lintels), support per-index param tables, and add Issue::PatternBroken/SymmetryResidual. Converts repetition from 'agent unrolls it' to 'system represents it' so downstream edits work.

### 3. [major] (foundation(0003)) — raised by Repetition, Archetypes
**Gap:** Reflection / bilateral MIRROR symmetry is unrepresentable: LatticeOrientation is the 24 PROPER rotations; reflections (the other 24 of the order-48 group) are excluded, so a left-hand wing cannot be a mirrored instance of a right-hand def. The most common architectural symmetry has no transform.
**Rec:** Widen the stored instance transform from the 24 proper rotations to the full 48-element signed-permutation group (add one handedness/reflection bit). Reflection is STILL an exact index permutation, preserving the lossless/byte-stable/sculpt-intact guarantee -- a one-field widening, not the deferred lossy-resample path. Cheapest while 0003 is Proposed. Sculpt-layer chirality of asymmetric ornament is the only caveat.

### 4. [major] (new-subsystem) — raised by Archetypes, Circulation, Structural, Site
**Gap:** Rooms / void / SPACE as a first-class object (a space graph). Explicitly deferred. But for churches/temples/houses/mills the SPACE is the primary design intent and the mass is residue; and circulation (egress, dead-end, room adjacency, 'walk narthex->apse') and vaults-over-a-bay all need a named void to attach to. Connectivity-over-joints is a part graph, not a space graph.
**Rec:** Promote the deferred space graph: segment resolved void into named space cells + portals, with SpatialQuery/diagnostics over IT (adjacency, portal connectivity, enclosed-volume, egress count, dead-end). A relational subsystem parallel to the joint graph (joints relate solids; this relates voids) and the shared substrate for traversability, vaults, and site datums.

### 5. [blocker] (new-subsystem) — raised by Circulation
**Gap:** Traversability / on-foot reachability under an agent body: 'can you actually walk from the gate to the keep top floor?' The only connectivity is the joint-graph path, which 0004 H6 deliberately chose to AVOID voxel flood-fill -- so two jointed parts say nothing about a navigable surface (stair landing into a solid wall, uncut stairwell, 1-block headroom). No walkable-surface model, agent body, gait validity, or unguarded-edge detection.
**Rec:** Add a traversability subsystem deriving a nav graph from RESOLVED occupancy (walkable = empty with solid below + headroom; edges within step/stride tolerance) with an AgentProfile. Add SpatialQuery::Reachable + Issue::Unreachable, gait diagnostics (StairTooSteep, HeadroomBelowMin, RunTooNarrow), and Issue::UnguardedEdge once a walkable map exists. Rides the resolved-grid read seam but is a new substrate (a derived nav/space graph) overturning the 'connectivity = joint-graph' stance -- confront deliberately.

### 6. [major] (foundation(0003)) — raised by Site
**Gap:** Terrain is READ-ONLY imported + producers may not read each other, but site/negative-space work is overwhelmingly about MODIFYING ground and defining elements RELATIVE to live grade: cut/fill, terrace, berm/rampart/glacis, terrain-relative basement floor, excavate-to-grade, retaining wall sized to the cut it holds. The frozen-terrain + no-coupling posture forecloses all of these.
**Rec:** Confront terrain writability before the kit is built: make terrain a first-class MUTABLE layer or allow a constrained controlled producer<->terrain coupling (GradeTo/Berm/Terrace/Excavate producers that sample GroundHeightAt internally to blend/tie-in). Pair with a terrain-relative DATUM concept (named site levels, terrain-relative Y). New diagnostics: terrain-tie-in-cliff, unbalanced-cut-fill. A foundation decision cheapest to revisit now.

### 7. [major] (new-subsystem) — raised by Structural, Circulation, Ornament
**Gap:** Structural SEMANTICS: load-path-to-ground, lateral thrust resolution (arches/vaults/domes/buttresses), and load-bearing-vs-decorative ROLE. The ONLY structural check is Unsupported = 'no block directly below' -- which fails real load reasoning AND false-positives on every arch, vault, dome, corbel, cantilever, and intentional ornament overhang, polluting the issue list the agent steers on.
**Rec:** For a PLANNER this may be out of scope -- decide explicitly. If wanted: a coarse load-path/thrust-balance pass over resolved occupancy + joint graph, with a part/def-level structural-role tag (LoadBearing|Decorative|Veneer|Tie). At minimum IMMEDIATELY fix the misleading stub: make the support check load-path-aware (trace diagonally/through arch action to ground/anchor) and add a 'decorative/overhang-exempt' flag so ornament and corbels stop false-firing.

### 8. [major] (kit/agent(0004)) — raised by Archetypes, Materiality, Structural
**Gap:** Frame-and-infill / discrete-member construction (half-timber fachwerk, post-and-beam longhouse, pole granary, timber pagoda frame, trusses, rib vaults). The compositor is mass-primary; WFC is material-only over FIXED occupancy so it cannot author the void RHYTHM between members. A timber frame is the opposite ontology: a sparse lattice of slender members + panel infill.
**Rec:** Add a STRUCTURAL-MEMBER producer family (a graph of beams/posts/braces with profiles -- reuses path-sweep machinery as a member network) plus an infill-panel producer, coordinated by a 'framed wall' composite module sharing one bay/rhythm parameter and registering members to opening connectors. State plainly that material-WFC cannot supply the occupancy rhythm. If structural reasoning is wanted, persist member endpoints as nodes rather than dissolving into unioned voxels.

### 9. [major] (kit/agent(0004)) — raised by Materiality
**Gap:** Walls/roofs are HOMOGENEOUS swept-profile+shell, but real construction differentiates material through-depth and across a unit tessellation: layered cladding skins (core+cavity+weather skin), masonry bonds/coursing, per-face finishes (render/tile-hang), thatch/shingle thickness+overhang, feature-relative dressings (quoins, stone jambs around openings). One block_id per voxel + Y-only banding fights every one of these.
**Rec:** Promote the wall PROFILE from a single section to a LAYERED cross-section (ordered (material,thickness) lamellae swept along the shared path) so multi-skin walls emit co-registered concentric skins (hidden bulk core falls out free). Add a coursing/bond producer tessellating into header/stretcher units with per-unit attrs + optional mortar seams. Extend HostedOn to carry a DRESSING (different-material reveal/jamb/sill/voussoir re-materializing with the opening) and add corner/edge-relative material keys for quoins. Consider a per-FACE material channel (foundation-touching) so thin finishes are surface data, not extra occupancy shells; treat fine mortar as a surface/shading attribute, not carved sub-voxels.

### 10. [major] (foundation(0003)) — raised by VS-specific, CAD/BIM
**Gap:** BlockAttrs is an undefined opaque 'rotation/variant flags' payload with no schema and no rule for how it transforms under LatticeOrientation. Most real VS blocks are STATEFUL (stair facing, log axis, door hinge/open, fence connectivity, slab half); a rotated instance keeps stale facings, neighbor-connection is uncomputed, and VS schematic export is lossy BY CONSTRUCTION -- a functional gatehouse exports as dumb stone.
**Rec:** Define BlockAttrs as a typed per-block-id state schema (orientation enum, variant flags, neighbor-connection bits) and specify how attrs COMPOSE with instance rotation (orientation must rotate like geometry). Add a neighbor-aware 'connection resolve' pass (state analogue of WFC). Add an optional sparse address-keyed block-entity side-table serialized ALONGSIDE the occupancy codec, anchored part-local. Then specify a VS-native schematic exporter round-tripping micro-block + block-state + entity data. Pin the attrs schema + rotation algebra now while 0003 is Proposed.

### 11. [major] (kit/agent(0004)) — raised by Circulation, Archetypes
**Gap:** Stair-vs-floor-opening coupling and multi-flight (spiral/switchback) vertical circulation: a stairwell hole is hosted on the floor but DRIVEN by the stair (a three-way dependency the transform-only joint graph can't express); moving the stair silently leaves the old hole and a stair butting a slab. No one-shot 'fit K risers between floor-Y and floor-Y+H' analogue to Span. Named stair producer is straight/dog-leg only; spiral is unaddressed.
**Rec:** Extend HostedOn from single-host to host+driver: a circulation penetration whose extent is computed from the stair's published swept-clearance ENVELOPE and re-cut when EITHER stair or floor changes. Add a vertical-circulation module that one-shot solves riser count/run/landing layout to bridge a height under a gait constraint (Span-along-Y), emitting flight+landing instances and the driven holes. Add HelicalStair/helical-sweep and Ladder producer variants (block-aligned connectors, no foundation change).

### 12. [major] (new-subsystem) — raised by Archetypes, Structural
**Gap:** Compound/ribbed curved VAULTS over interior spaces (barrel/groin/rib/fan/pendentive, upturned pagoda eaves). Roofs are straight-skeleton hosted on the OUTER footprint; a groin/rib vault is CSG of curved surfaces conformed to an interior BAY and is the ceiling of a space -- the skeleton doesn't reach it and the host is wrong.
**Rec:** A VAULT producer family parameterized by a bay footprint + springline height, hosted on a room/bay rather than the building footprint. Depends on the space/bay graph; net-new producer subsystem distinct from the skeleton roof. Pendentives/squinches/drums are expressible as geometry; only their structural meaning is lost, which folds into the thrust gap.

### 13. [major] (kit/agent(0004)) — raised by CAD/BIM
**Gap:** Recursive shape-grammar / split-and-repeat facade subdivision (CGA: split wall->floors->bays->window+pier, comp onto faces, recurse) and the general integer split-with-remainder / divisibility problem (tile L into N bays + distributed remainder, keep openings centered). The kit is a flat registry of one-shot expand() modules; WFC can't decide occupancy; only the single parapet-rhythm instance of split-with-remainder is solved.
**Rec:** Add a recursive subdivision/grammar KitModule kind (split along axis into ratios/repeats with remainder, comp-onto-faces, bounded recursion -> deterministic Intents) and promote a SHARED integer-subdivision utility (fixed|proportional|repeat-with-remainder|centered) callable by kit AND queryable by the agent, beside SubBlockSnap, with a divisibility/alignment diagnostic. Subsumes L-system recursion and the curtain-wall panel grid.

### 14. [major] (kit/agent(0004)) — raised by Repetition, Ornament, Materiality
**Gap:** Coursing/bond/merlon PHASE registration across junctions and around corners. Material banding gets world-Y phase free, but that key is Y-only: horizontal/along-path/around-corner rhythm has no shared key, so each part's PatternAlong has an independent phase origin and a junction risks a half-merlon/half-brick seam. 0004's own P1 probe flags this as OPEN.
**Rec:** Generalize the world-Y band key into a shared world-space PHASE FIELD over all three axes (or a kit-level 'coursing datum' modules sample), so along-path/around-corner rhythm keys to a shared origin. Couple with an anchored/registered pattern primitive (fit-integer-count-between-anchors, phase locked to corners/openings, continuity across mitered runs).

### 15. [minor] (new-subsystem) — raised by Site
**Gap:** Water as a FILLED VOLUME with a level surface (moats/ponds/reservoirs/flooded cellars): water settles to a horizontal plane at elevation Y across an irregular basin. The pure solid-occupancy cell has no fill-level concept; it depends on the resolved basin so it's neither a static producer nor hand-painted blocks surviving basin edits.
**Rec:** A derived 'flood-fill to level Y within a bounded basin' post-resolve pass (containment-aware fill reachable within the containing walls up to a chosen plane), recomputed when the basin changes, plus Issue::BasinNotWatertight. A new compute pass over resolved occupancy (a producer reading the composite), adjacent to the terrain-coupling problem.

### 16. [major] (foundation(0003)) — raised by CAD/BIM, Site
**Gap:** Levels / grids / DATUMS as named global reference geometry many parts attach to ('move Level 3 up and everything on it follows'; columns on grid A-3). Joints are part-to-part; no shared datum propagates. Also no family/TYPE tier or per-instance PARAM overrides -- an instance is transform-only, forcing a def fork per size variant.
**Rec:** Add a Datum primitive (level plane / grid line / work axis) as scene-owned named reference geometry + a HostedOnDatum relationship (reuses NodeId/Joint machinery). Separately bend 'instance = transform only' to let an Instance carry a KitParams override bag and/or add a named TYPE tier on a Def -- CAD/BIM treat this as table stakes and the design already bent an analogous purity rule (assembly_overrides). Both are foundation decisions cheapest now.

### 17. [major] (new-subsystem) — raised by Ornament, CAD/BIM
**Gap:** Host-EDGE/SURFACE substrate at chisel resolution + persistent topological naming. Producers are pure and connectors are block-snapped point/face frames, so ornament that must FOLLOW a host edge/silhouette (raking cornice up a gable, string course wrapping a round tower, molding along an arch, surface-mapped relief/heraldry) has no anchor; overrides/hosted-features bind to brittle voxel ADDRESSES, not 'roof.ridge' -- the classic CAD topological-naming problem; plus corner miter/return is the wall-junction problem replayed sub-block.
**Rec:** Introduce a semantic/persistent reference scheme for DERIVED geometry (named feature outputs: roof.ridge, wall.top_edge, skeleton.face[k]) surviving regeneration, plus a chisel-resolution EdgeCurveOf/SurfaceFrameAlong query and a HostedOn-follows-redrape relationship (the molding analogue of GroundHeightAt+footing). Grow connectors to 1D edge-path connectors. This substrate is what hosted features/penetrations/dressings quietly assume; it also unblocks a real in-part feature tree (extend the closed 2-arm Layer enum so features consume prior features' topology) and a sub-block miter/return solver.

### 18. [major] (new-subsystem) — raised by Materiality
**Gap:** No stochastic-OCCUPANCY engine for weathering / damage / ruins / partial collapse / decay gradients. WFC is hard-locked to material-only over FIXED occupancy, so it cannot remove blocks; occupancy removal is only deterministic Subtract or hand force-off. Ruins are a first-class want for a VS castle/fort planner.
**Rec:** Add a seeded DECAY/EROSION subsystem (a producer or override-generating tool) taking an exposure/age field + a target-material filter, emitting stochastic force-off (occupancy removal) AND material reclassification (stone->mossy-stone), ALONGSIDE WFC. This deliberately steps outside the 'WFC never moves geometry / occupancy fixed before fill' invariant -- name that. A noise/irregular subtractive Cave/Erode producer for natural karst fits the producer registry cleanly and is the one negative-space want already structurally accommodated.

### 19. [minor] (kit/agent(0004)) — raised by Site
**Gap:** Drainage/flow gradient: sewers/aqueducts/culverts must slope continuously to an outfall. Connectivity is topological only; nothing validates a monotonic descent or detects a back-pitch/sump.
**Rec:** A gradient/flow diagnostic over a conduit void floor (sample floor-Y along the path, assert monotonic descent to a declared outfall, flag local minima as Issue::DrainageBackpitch). Kit/agent-layer check on the existing region read; optionally a gradient-enforcing path drape like GroundHeightAt.

### 20. [minor] (kit/agent(0004)) — raised by Ornament, CAD/BIM
**Gap:** Column ORDERS / cross-parameter derivation and global driving-parameter expressions (capital height = N modules; tower_height = wall_height + 3; all merlons.width = parapet.module_width). KitParams are independent per-module with no expression layer; the joint solver constrains transforms, not params.
**Rec:** A parameter-relation/expression layer in the kit (named params + simple expressions evaluated into SetParam intents) so orders rescale consistently and cross-element parametric intent works without the LLM doing all arithmetic externally. Pure additive on top of the existing Intent door.

### 21. [minor] (kit/agent(0004)) — raised by VS-specific
**Gap:** Per-material COST accounting / bill-of-materials and world-grid REGISTRATION on export. Survival VS builders budget materials; nothing aggregates resolved occupancy into per-block_id / chisel-op counts. The store's quantized floating origin is decoupled from any world-meaningful origin, so a world-origin export contract and 16-grid phase agreement are undefined.
**Rec:** Add SpatialQuery::MaterialTally/BillOfMaterials over a region (pure consumer of the read path). Define a world-origin export contract (a build-anchor block coord mapping to a VS world coord) and assert chisel detail is phased to the block-local 16-grid before round-trip.

### 22. [minor] (defer) — raised by Circulation, Archetypes, VS-specific
**Gap:** Movable / STATEFUL circulation & mechanisms (drawbridge up/down, portcullis, lift, windmill cap yaw, doors that open). The model has exactly one resolved occupancy per scene; reachability/closure are inherently state-conditional. Even the STATIC pose of a windmill cap at 37deg needs the arbitrary-axis fix.
**Rec:** Keep the planner explicitly STATIC: model function only as inert ANNOTATION (block_id='door', attrs='hinge-left/closed', entity side-table for contents) carried for EXPORT fidelity so VS supplies behavior. Fold static movable POSES into the arbitrary-axis instance fix. Live kinematics/state-machine/tick/signal simulation is a categorically different subsystem -- defer and document the boundary, do not bolt it into the three-tier static model.

## Blind spots (no lens covered)
No lens covered these whole categories, and several matter: (1) ACOUSTICS / SIGHTLINES / FIELDS-OF-FIRE -- defensive architecture is defined by lines of fire (arrow-slit cones, machicolation drop zones, dead ground), theaters/churches by sightlines/acoustics; this is a visibility/ray field over resolved occupancy, a sibling of the missing light-field analysis, and no lens raised it. (2) COLLABORATIVE / VERSIONING / MERGE & PROVENANCE -- the Intent journal gives replay, but nothing addresses multi-author merge, branching designs, diffing two scenes, or conflict resolution of concurrent edits; for a tool people share builds with, a real omission. (3) UNITS / SCALE / REAL-WORLD DIMENSIONING -- everything is blocks/voxels; no real-world unit binding, dimension annotation, or measurement/labeling layer that CAD users and dimension-conscious VS builders expect. (4) HUMAN DIRECT-MANIPULATION EDIT ERGONOMICS -- the journal is replayable but the whole stack is evaluated as agent-driven; whether a human can author, multi-select, group, and undo at sane granularity is unexamined. (5) AGENT-LOOP FAILURE MODES -- the perceive->diagnose->correct loop is described for well-posed intents, but contradictory constraints, unsatisfiable joint sets, and agent hallucination of non-existent kit modules (the failure taxonomy of the AUTHORING channel itself) are unexamined. (6) PERFORMANCE/MEMORY BUDGET OF THE NEW ANALYSIS SUBSYSTEMS -- the sweep proposes ~6 new subsystems (traversability, space graph, structural, terrain-coupling, decay, topological naming) all reading resolved occupancy at >10k-XZ scale; no lens asked whether running these region-scoped analyses concurrently fits the streaming/worker-pool budget, which is where they will actually collide. (7) TEXT / TYPOGRAPHY / SYMBOLIC CONTENT beyond heraldry -- inscriptions, maps, banners authored from a glyph/asset library, not hand-sculpted.

## Reassuringly covered
The producer/voxel/three-tier spine is genuinely strong and the right altitude -- every lens independently concluded there is NO outright model-breaker for geometry+placement. Specifically well-covered: the wall = profile-swept-along-3D-path producer is a workhorse that generalizes for free to cornices, string courses, copings, bridge decks, aqueduct channels, moats, roads, tunnels/sewers (same sweep, different profile/path). The SHELL op + mass<->shell continuum eliminates a wall-vs-building type split and serves cellars/crypts underground exactly as above grade. Arbitrary-ANGLE one-off geometry is cleanly answered (angle is a producer param, voxelized/staircased, never a transform limit). The Fusion def/instance + edit-propagates model, the n-ary joint graph with topological propagation/cycle-fixpoint/over-constraint detection, HostedOn auto-recut, ordered later-wins compositor, stable NodeId/DefId/JointId arenas, the typed block-snapped connector substrate, the straight-skeleton for roofs and plan junctions, and the demoted-deterministic material-only WFC keyed to world-Y bands are all judged solid. Material banding, terrain-DRAPING (GroundHeightAt + WallBaseGap + footing sub-producer), the block<->chisel two-scale seam matching VS micro-blocks exactly, the anisotropic absolute-i64 streaming store with full-Y-column residency (a happy path for deep shafts), and the deterministic/replayable Intent journal are all correctly placed. The footprint->massing->facade->detail prior-art map and the connector/socket system are exactly the CAD-adjacent substrates the harder gaps build on.

## Top recommendations

 - FOUNDATION, do now while 0003 is Proposed -- widen the instance transform from the 24 proper rotations to the full 48-element signed-permutation group (add one reflection/handedness bit). It stays an exact, lossless, byte-stable index permutation, costs almost nothing, and unlocks bilateral symmetry, the most common architectural symmetry, otherwise outside the stored transform entirely.
 - FOUNDATION, do now -- pin down BlockAttrs as a typed per-block-id state schema AND specify how it composes with LatticeOrientation (orientation must rotate with the geometry). Load-bearing: without it, rotated stairs/logs/doors keep stale facings and VS schematic export is lossy by construction. Pair with the decision to carry a sparse block-entity side-table and a world-origin export contract.
 - FOUNDATION, decide now -- confront the two stated invariants the sweep most strained: (a) terrain READ-ONLY + no producer<->producer coupling (blocks all cut/fill/terrace/terrain-relative work -- allow a controlled terrain-querying coupling + a terrain-relative datum), and (b) instance = transform-only (blocks family/TYPE tiers and per-instance params CAD treats as table stakes -- allow an instance param-override bag). Both are cheap to relax now, expensive after the kit is built on the old assumptions.
 - KIT/0004, the highest-leverage single primitive -- build the parametric PATTERN/REVOLVE producer that arrays a sub-geometry at arbitrary angles FROM THE FIELD while exposing stable per-member connectors and indices. This one hybrid resolves the deepest cross-lens tension (radial/curved/helical repetition of reusable parts) AND is the substrate for per-member overrides, editable arrays, bay repetition, and curtain-wall grids.
 - IMMEDIATELY fix the misleading Unsupported/Overhang stub -- it false-positives on every arch, vault, dome, corbel, cantilever, and intentional ornament overhang, actively polluting the issue list the agent steers on. At minimum add a 'decorative/overhang-exempt' role flag and make support tracing load-path-aware; defer the full statics/thrust subsystem unless structural validity is declared in-scope.
 - Make a deliberate SCOPE RULING on static-vs-functional and sequence the new analysis subsystems. Declare the planner STATIC (function = inert annotation for export), and promote the deferred SPACE GRAPH first, since traversability, vaults, circulation egress, and site datums all depend on void-as-first-class -- it is the shared substrate for the most blockers.
